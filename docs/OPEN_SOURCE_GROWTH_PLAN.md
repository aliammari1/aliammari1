# Open Source Growth Plan

This plan converts the repository review into an actionable backlog for the GitHub profile repository.

## Quick wins

- Make the profile README a project launcher with your strongest public repositories above the fold.
- Add concise one-line value propositions for each featured project.
- Add badges for portfolio, LinkedIn, email, and selected technologies.
- Add a `Now building` section for active projects.
- Add a `Featured open-source work` section that links to live demos, screenshots, and PR-friendly repos.

## Bugs and bad practices to watch

- Too many unfocused badges or widgets can make the profile noisy.
- Old or private project links can reduce trust.
- Missing calls to action make it harder for visitors to star or contribute.
- Generated stats cards can slow rendering or fail intermittently.

## Star growth strategy

1. Route visitors to the top 3 repositories you want starred.
2. Add short demo GIFs or screenshots for the most visual projects.
3. Link to contribution guides and `good first issue` labels.
4. Use consistent naming and descriptions across profile and repo READMEs.
5. Add a portfolio link with project case studies.

## Trending-library opportunities

- Feature AI/data projects using Polars, Pydantic-AI, Data-Formulator, MarkItDown, and LangExtract where relevant.
- Add a small `Tech radar` section showing tools you are exploring.

## Suggested next PRs

- Rewrite the profile README around featured projects and clear calls to action.
- Add a project matrix with repo, stack, demo, and contribution status.
